Queries

facets

Description

Returns facets category

Response

Returns a Facets

Arguments
Name Description
query - String Deprecated. Use fullText instead. Default = ""
fullText - String Search term. It can contain any character. Default = ""
map - String Deprecated. Use selectedFacets instead. Default = ""
selectedFacets - [SelectedFacetInput]

List of the selected facets

The order in which the terms appear is not relevant to the search.

You can also repeat the same facetKey several times for different values. Default = []
hideUnavailableItems - Boolean If true, uses isAvailablePerSalesChannel_ parameter on query with segment's sales channel. Default = false
removeHiddenFacets - Boolean If true, remove hidden facets from the result. Default = false
behavior - String If Static, ignores SpecificationFilters received on the map and query when returning the facets available, which makes the facets never change. Default = "Static"
operator - Operator Indicates how the search-engine will deal with the fullText if there is more than one word. Set and if the returned products must have all the words in its metadata or or otherwise.
fuzzy - String Indicates how the search engine will correct misspelled words by using fuzzy logic. It can be a number representing the max number of misspelled letters, or the string auto suggesting that the search-engine should set this value by itself.
searchState - String As fuzzy and operator, it controls the search state, but it is for general purposes. This field allows the search engines to apply features that are not handled by the other fields. The possible values in this field are defined by each search engine.
from - Int Pagination item start
to - Int Pagination item end
categoryTreeBehavior - CategoryTreeBehavior Determines the behavior of the category tree. Default = default
initialAttributes - String Initial attributes (based on the initialMap parameter)

Example

Query

  $query: String,

      ...SearchBreadcrumbFragment
    }
  }
}

productSearch

Description

Lists the products for a given query.

Response

Returns a ProductSearch

Arguments
Name Description
query - String Deprecated. Use fullText instead. Default = ""
fullText - String Search term. It can contain any character. Default = ""
map - String Deprecated. Use selectedFacets instead. Default = ""
selectedFacets - [SelectedFacetInput]

List of the selected facets

The order in which the terms appear is not relevant to the search.

You can also repeat the same facetKey several times for different values.
category - String Deprecated. Use selectedFacets instead. Default = ""
specificationFilters - [String] Deprecated. Use selectedFacets instead.
priceRange - String Filter by price range. e.g.: {a} TO {b} - {a} is the minimum price "from" and {b} is the highest price "to". Default = ""
collection - String Deprecated. Use selectedFacets instead. Default = ""
salesChannel - String Filter by availability at a specific sales channel. e.g.: salesChannel:4 if want filter by available products for the sales channel 4. Default = ""
orderBy - String Order by a criteria. OrderByPriceDESC/OrderByPriceASC, OrderByTopSaleDESC, OrderByReviewRateDESC, OrderByNameASC/OrderByNameDESC, OrderByReleaseDateDESC, OrderByBestDiscountDESC, OrderByScoreDESC If you want to sort by a specification, use the format {specification key}:{asc|desc}. For example: "pricePerUnit:asc" or "size:desc" (this only works on vtex.search-resolver@1.x). Default = "OrderByScoreDESC"
from - Int Pagination item start. Default = 0
to - Int Pagination item end. Default = 9
hideUnavailableItems - Boolean If true, uses isAvailablePerSalesChannel_ parameter on query with segment's sales channel. Will override any given salesChannel arg. Default = false
simulationBehavior - SimulationBehavior If you want faster searches and do not care about most up to date prices and promotions, use skip value. Default = default
productOriginVtex - Boolean Each search engine has its own database, but this database might not have all the product information like clusterHighlights or productClusters. As an alternative, the search engine may use the VTEX API to complete this information by setting this field to true. Default = false
operator - Operator Indicates how the search-engine will deal with the fullText if there is more than one word. Set and if the returned products must have all the words in its metadata or or otherwise.
fuzzy - String Indicates how the search engine will correct misspelled words by using fuzzy logic. It can be a number representing the max number of misspelled letters, or the string auto suggesting that the search-engine should set this value by itself.
searchState - String As fuzzy and operator, it controls the search state, but it is for general purposes. This field allows the search engines to apply features that are not handled by the other fields. The possible values in this field are defined by each search engine.
options - Options Search options that customize the search result.

titleTag - String Title used in the title's tag
metaTagDescription - String String to be used in the <meta name="description"... tag
breadcrumb - [SearchBreadcrumb] Use the breadcrumb from the facets query instead.
canonical - String
suggestion - SearchSuggestions Use the suggestion query instead.
correction - SearchCorrection Object that indicates if the term was misspelled and suggests a possible correction.
operator - Operator Indicates how the search-engine dealt with the fullText when there is more than one word.\n * and - It means that the products contains all the words in the query.\n * or - It means that the results will contain at least one word from the original search query. If and was not possible, or will be the fallback.
fuzzy - String Indicates how the search engine corrected the misspelled word by using fuzzy logic. It can be a number representing the max number of misspelled letters, or the string auto suggesting that the search-engine should set this value by itself.
searchState - String As fuzzy and operator, it controls the search state, but it is for general purposes. This field allows the search engines to apply features that are not handled by the other fields. The possible values in this field are defined by each search engine.
banners - [SearchBanner] Use the banners query instead.
redirect - String Returns the redirect url set for the given query.
Example
{
  "products": [Product],
  "recordsFiltered": 987,
  "titleTag": "xyz789",
  "metaTagDescription": "xyz789",
  "breadcrumb": [SearchBreadcrumb],
  "canonical": "abc123",
  "suggestion": SearchSuggestions,
  "correction": SearchCorrection,
  "operator": "and",
  "fuzzy": "abc123",
  "searchState": "abc123",
  "banners": [SearchBanner],
  "redirect": "abc123"
}

Description

Object that indicates if the term was misspelled and suggests a possible correction.

Field Name Description
